GitHub自定义域名的绑定解析

4月 3 2015

弄这个Github博客花了半天的时间，域名解析上栽了大部分时间完全没有想到。主要是在看到“简书”上的一个教程ping域名的时候ping的是pages.github.com这个网址的IP地址，结果在dnspod上配置后一直无法访问。

刚开始以为是解析还没有生效，过了2小时后还不行，害的我一直认为是CNAME的问题，反反复复改了半天。最后才想到是IP地址的问题，看来撸多了，智商也是越来越不行了。你要ping的是你的GitHub博客二级域名，比如我的就是shupande.github.io

得到地址后，再进入dnspod进行设置，添加a记录。

当然还有在GitHub仓库中建立CNAME文件填入你的域名，注意这里不用添加www,本地硬盘目录下面也要添加一个不然同步的时候文件会消失。然后就大功告成了。我的解析更改后就立即生效了，不得不夸dnspod的效率真的非常高！
